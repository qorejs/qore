## 📊 版本发布流程

### 标准发布流程

```
1. 分支创建
   └── git checkout -b release/vX.Y.Z

2. 版本更新
   └── 更新 packages/*/package.json

3. 变更日志
   └── 更新 CHANGELOG.md

4. 测试验证
   ├── pnpm test
   ├── pnpm test:coverage
   └── pnpm bench

5. 构建验证
   └── pnpm build

6. 团队评审
   └── PR 评审 + 验收

7. 发布
   ├── npm publish
   └── GitHub Release

8. 发布后
   ├── 官网更新
   ├── 社区公告
   └── 反馈收集
```

### 紧急发布流程 (Hotfix)

```
1. 问题确认
   └── 严重 Bug / 安全漏洞

2. 快速修复
   └── 创建 hotfix 分支

3. 最小化测试
   └── 只测试修复相关功能

4. 快速评审
   └── 核心团队成员评审

5. 发布
   ├── npm publish (PATCH 版本)
   └── 安全公告 (如适用)

目标：24 小时内完成
```
